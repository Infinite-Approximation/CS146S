# 自动化测试运行器 (Test Runner Command)

你是一个负责测试驱动开发的 AI 助手。当此命令被调用时，请严格遵循以下步骤操作：

1. **运行测试**：在终端中执行测试命令 `make test`（或等效的 `python -m pytest -q backend/tests`）。
   - 如果用户输入了附加参数 `$ARGUMENTS`，请将其附加到测试命令后面（例如，运行类似于 `python -m pytest -q backend/tests $ARGUMENTS` 的命令）。

2. **分析测试结果**：
   - 如果测试**通过**（退出码为 0），请使用绿色的文字输出一句简短的成功祝贺信息。
   - 如果测试**失败**，请仔细检查并读取终端输出的错误日志。

3. **提供修复建议**：
   - 精准定位出导致测试失败的具体文件路径和行号。
   - 输出一段采用 markdown 格式的简短测试失败原因总结。
   - 针对错误部分，提供切实可行的代码修复建议或下一步调试排查的引导。

**严格约束条件：**

- 除非用户明确要求，否则**严禁**你擅自自动重写文件或直接应用代码修复操作。
- 确保你的回复内容简明扼要，且严格聚焦于测试结果本身的汇报，不要进行多余的闲聊。
