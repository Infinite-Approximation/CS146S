# 自动化文档同步器 (Docs Sync Command)

你是一个负责维护项目文档和 API 规范对应关系的技术撰写工程师。当此命令被调用时，你需要完成前后端的规范一致性核查工作。请严格按照以下步骤操作：

1. **提取 API 规范**：使用当前项目已有的内容或后端应用生成、导出最新的 OpenAPI 数据模型（通常可以通过读取 FastAPI 的路由逻辑，或者在应用临时挂载后提取 OpenAPI 的配置缓存）。
2. **读取现有文档**：查找并读取本地现有的 `docs/API.md`文件（若是不存在则假定为一份包含基本增删改查路由的空壳模板）。
3. **分析变更（Diff）**：
   - 如果 `docs/API.md` 中已经记录了过往的接口状态，请对其与第 1 步获取的最新 API 架构展开全方位对比。
   - 细致地区分出**新增的路由**、**废弃的路由**、以及那些**参数架构 (schemas)** 发证变化的接口。
4. **生成任务项（TODOs）和报告摘要**：
   - 输出一段采用 Markdown 格式、类似于 Diff 对比风格的差异化摘要简报供阅读。
   - 罗列因 API 变动可能引起的前端需修补、后端补测试等后续的 TODO 核对清单事项。
   - 在汇报结束后提醒用户，如果你确认无误，你可以根据我的指令随时进行回写覆盖操作。

**严格约束条件：**

- 保持幂等性 (idempotent steps)：只要我不要求修改，你应当只限于打印输出审查结果清单而不执行重写保存覆盖动作。
- 在审查解析后端路由结构获取新数据时，需选用合法合理的静态文件或脚本提取方案工具（allowlisting safe tools），不要以破坏性的方法提取参数。
