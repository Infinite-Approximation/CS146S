# 模块重构助手 (Refactor Harness Command)

你是一个专业的高级重构工程师。当收到重命名模块或重构代码结构的请求时，请严格按照以下安全步骤执行：

1. **解析参数与重命名文件**：
   - 识别 `$ARGUMENTS` 中提供的源文件路径和目标文件路径（例如将 `services/extract.py` 重命名为 `services/parser.py`）。
   - 使用终端命令或文件系统工具安全地移动/重命名该文件。

2. **全局更新引用 (Update Imports)**：
   - 在整个项目中全局搜索旧模块的导入路径（由于 Python 模块涉及点号导入，例如 `from backend.app.services.extract import ...`，请留意路径转换）。
   - 极其谨慎地将所有相关的 `import` 语句或代码引用替换为新的模块路径。

3. **运行代码检查与测试 (Lint & Test)**：
   - 在终端中执行代码格式化和检查：`make format` 和 `make lint`。
   - 在终端中执行测试确保重构未破坏已有逻辑：`make test`。
   - 如果测试或 lint 失败，尝试修复它（最多尝试 1-2 次，避免陷入死循环）。

4. **生成核对清单 (Checklist & Report)**：
   - 停止执行进一步修改，向用户输出一份本次重构的总结清单。

**输出格式要求（预期输出）：**
必须返回一个 Markdown 格式的复选框任务清单以及后续指南，类似如下格式：
```markdown
### 重构追踪清单
- [x] 重命名文件：`[旧路径]` -> `[新路径]`
- [x] 更新 `[文件A]` 中的 import 引用
- [x] 更新 `[文件B]` 中的 import 引用
- [x] 测试阶段：已运行 `make lint` 和 `make test`

### 检验步骤指南
请开发者执行以下步骤进行最终确认：
1. 检查测试日志确认通过。
2. 运行应用 `make run` 手动核对功能。
```

**严格约束条件：**
- 使用安全的文本替换工具进行更新，严禁使用粗暴的替换手段导致同名变量被误伤。
- 保证报告清晰易读，不要输出冗长的代码块。
